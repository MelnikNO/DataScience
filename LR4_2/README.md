## Модели, рассматриваемые в ноутбуке

### 1. Baseline модели
- **Naive Forecast** - прогноз = последнее значение
- **Seasonal Naive** - прогноз = значение из прошлого сезона
- **Moving Average** - прогноз = среднее за окно

### 2. Статистические модели
- **ARIMA(p,d,q)** - AutoRegressive Integrated Moving Average
- **SARIMA(p,d,q)(P,D,Q,s)** - Seasonal ARIMA
- **Holt-Winters** - Exponential Smoothing

### 3. Машинное обучение
- **Linear Regression** - линейная регрессия с лаговыми признаками
- **Random Forest** - ансамбль деревьев решений
- **Gradient Boosting** - градиентный бустинг

### 4. Глубокое обучение (опционально)
- **LSTM** - Long Short-Term Memory нейронная сеть

## Метрики качества

В ноутбуке используются следующие метрики:

1. **MAE (Mean Absolute Error)** - средняя абсолютная ошибка
2. **MSE (Mean Squared Error)** - среднеквадратичная ошибка
3. **RMSE (Root Mean Squared Error)** - корень из MSE
4. **MAPE (Mean Absolute Percentage Error)** - средняя абсолютная процентная ошибка
5. **R² (Coefficient of Determination)** - коэффициент детерминации

Все метрики рассчитываются автоматически для каждой модели и сводятся в таблицу для удобного сравнения.

## Backtesting

Ноутбук включает реализацию **Walk-Forward Validation** с использованием `TimeSeriesSplit` из scikit-learn. Это позволяет:
- Имитировать реальное использование модели
- Получить более надежную оценку качества
- Избежать переобучения
- Понять стабильность модели во времени

## Типичные ошибки и как их избежать

### 1. Нарушение временной последовательности
**Ошибка:** Перемешивание данных при разделении на train/test  
**Решение:** Всегда сохраняйте временной порядок!

### 2. Data Leakage
**Ошибка:** Использование будущей информации для прогноза прошлого  
**Решение:** Создавайте лаги и признаки только на основе прошлых данных

### 3. Игнорирование стационарности
**Ошибка:** Применение ARIMA к нестационарному ряду  
**Решение:** Проверяйте стационарность и применяйте преобразования

### 4. Переобучение
**Ошибка:** Слишком сложная модель показывает отличные результаты на train, но плохие на test  
**Решение:** Используйте backtesting и регуляризацию

### 5. Неправильная оценка
**Ошибка:** Оценка модели только на одном test split  
**Решение:** Используйте TimeSeriesSplit для кросс-валидации
